# v0.9.9

Reliability fixes for the workflow engine.

## ⚠️ Behavior change

- **`pallium workflow run` now exits non-zero when a run fails entirely.**
  Previously it exited 0 even when every agent failed (e.g. the provider was
  out of quota), which could mask a total failure in a CI script. A run that
  completes with at least one successful agent still exits 0. If a script
  relied on the old always-zero exit code, update it.

## Fixed

- **Completed edit work is no longer discarded when structured output fails
  schema validation.** When an edit agent produced a patch but its final JSON
  failed the declared `schema`, the patch was silently dropped and the run
  failed — the edit vanished. The patch is now preserved and applied (and is
  recoverable via `pallium workflow apply`), and the schema failure is reported
  in the run's failures list. Only the structured output fails, not the work.
  Read-only agents are unchanged (their structured output is the deliverable).
- **Honest provider errors.** A failed provider command now surfaces the
  meaningful error line (usage/quota/rate-limit/auth) from its output instead
  of an opaque `signal: killed: Reading additional input from stdin`.
- **Workspace index is shared across git worktrees.** `pallium workflow
  preflight` and the analysis commands run from a linked worktree of an
  already-indexed repo now reuse that repo's index instead of reporting
  "not indexed" and forcing a redundant reindex.
