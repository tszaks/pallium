# v0.9.9 — Reliability fixes

A focused release closing three real reliability gaps in the workflow engine.

## Fixed

- **Completed edits are never discarded again.** If an edit worker's final
  report failed validation, its actual code changes used to be silently
  dropped along with it. The edit is now preserved and applied; only the
  report itself is flagged as a failure.
- **Honest error messages.** A failed worker now reports the real reason
  (quota, rate limit, authentication) instead of an unhelpful generic error.
- **Faster repeat scans.** Running Pallium from a linked git worktree of an
  already-indexed repo now reuses that index instead of redoing the work.

## Behavior change

`pallium workflow run` now exits with a non-zero status if every worker in
the run failed. Previously it always exited 0, which could hide a total
failure in a CI script. A run with at least one successful worker still
exits 0.

## Install

```bash
npm install -g pallium
pallium version
```
