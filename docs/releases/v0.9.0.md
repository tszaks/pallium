# v0.9.0

Pallium v0.9.0 adds the first release of Console coordination and Pallium-owned
session control.

## Added

- `pallium console` coordination for live session visibility, manifests,
  handoffs, file claims, action requests, authority gates, and review gates.
- `pallium console run` to spawn a foreground PTY-backed owned session.
- `pallium console run --background` to spawn a detached owned session.
- `pallium console read` to read captured owned-session logs.
- `pallium console owned list` and `pallium console owned show` to inspect
  owned-session metadata.
- `pallium console interrupt` to interrupt a Pallium-owned process group.

## Hardened

- Owned-session IDs are validated before use in default log paths.
- Owned-session working directories are resolved and validated before launch.
- Missing owned-session updates now fail instead of silently succeeding.
- JSON output now includes `exit_code: 0` for successful owned sessions.
- `read --tail` handles trailing newlines correctly.
- Interrupt handling targets the owned process group, with direct process
  fallback.

## Boundary

This release controls sessions that Pallium spawned itself. It does not inject
commands into arbitrary existing Codex or Claude Code sessions.
