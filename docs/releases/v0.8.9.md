# v0.8.9 — Claude Code session memory

Pallium's session memory now covers Claude Code, not just Codex.

## What's new

- **Claude Code transcript indexing.** Pallium indexes Claude Code sessions
  from `~/.claude/projects/`, alongside existing Codex support.
- **Choose which provider to index.** `pallium sessions index --provider
  all|codex|claude` and `--claude-home` for a non-default Claude transcript
  location.
- **Richer session details for Claude.** Indexed Claude sessions capture
  titles, working directory, git branch, model, messages, tool calls,
  commands, file paths, and errors.

## Install

```bash
npm install -g pallium
pallium version
```
