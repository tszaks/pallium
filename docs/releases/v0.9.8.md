# v0.9.8 — Any model, one command

Model selection becomes universal, and Pallium gets a true one-command entry
point.

## What's new

- **`pallium start "<task>"`.** Describe what you want in plain language, and
  Pallium generates and runs a workflow for it automatically — no need to
  know the lower-level commands. Falls back to a safe, deterministic template
  if generation fails.
- **Every model, everywhere.** Worker selection, workflow generation, and
  `pallium start` all now resolve the same way: adopt whichever agent is
  steering, fall back to Codex only when nothing else is detected. No model
  is hardcoded anywhere in Pallium.
- **Bring your own model.** A documented, minimal wrapper contract lets any
  model's CLI power Pallium — no code change required.

## Install

```bash
npm install -g pallium
pallium version
```
