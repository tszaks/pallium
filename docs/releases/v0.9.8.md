# v0.9.8

Pallium v0.9.8 makes model selection universal: any model can power the
workers, any agent can steer Pallium, and no model is hardcoded anywhere.

## Added

- `pallium start "<task>"` — the golden path. From a natural-language task it
  generates a repo-scoped workflow using the current model and runs it, so an
  agent can describe what it wants in one command without knowing the
  lower-level preflight/generate/run commands. `--dry-run` previews the
  resolved workflow; `--workflow`, `--style`, and `--json` are supported. When
  no provider can generate (or returns an invalid script) it falls back to a
  deterministic template and says so, rather than masking a broken provider.

## Changed

- Every model invocation — worker agents, `workflow generate --llm`, and
  `pallium start` — now resolves its provider through one path and adopts the
  agent steering Pallium. Running from inside Claude Code uses Claude for
  generation and workers automatically; Codex is only the fallback when no
  steering agent is detected. `workflow generate --llm` is no longer
  Codex-only.
- Provider process invocation is consolidated: the codex binary is invoked
  from exactly one file, and a build-failing test guards against reintroducing
  a hardcoded provider exec elsewhere.
- `providers/README.md` documents a minimal wrapper so any model CLI can power
  Pallium via `PALLIUM_WORKFLOW_PROVIDER_<NAME>_COMMAND`, no code change.
