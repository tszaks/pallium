# v0.9.10

Pallium v0.9.10 adds Agent Teams: a lead plus independent named peer agents
that coordinate over a shared task board and mailbox, each with a real
persistent session that survives across turns — and across the steering
process being killed.

## Added

- `pallium team start|spawn|tasks|send|inbox|nudge|status|run|approve|stop|attach`
  — turn-based teammates. Unlike a workflow's `parallel()`/`pipeline()` (still
  the right tool for fan-out work that doesn't need peer-to-peer
  coordination), a team is for work that benefits from PEERS reasoning
  independently and messaging each other: parallel review from different
  angles, a research-then-synthesize pattern, adversarial debate.
- Each teammate's identity is a durable SQLite row, not a process: turns are
  separate provider CLI calls that resume the SAME native conversation.
  Claude teammates get a Pallium-minted session id (`--session-id` on the
  first successful turn, `--resume` after); codex teammates are tracked via
  `codex exec --json`'s own thread id. `pallium team run <id>` is bounded
  like `workflow run` — it drives turns until the team converges (no mail,
  nudge, or claimable work left) and exits, no daemon. `pallium team attach
  <id>` reconnects to an existing team from any process, any agent, later —
  including after a hard kill mid-turn.
- Coordination (messaging, task claim/complete, going idle) happens through
  one structured end-of-turn decision Pallium parses and applies, not
  mid-turn tool calls — uniform across providers regardless of sandbox model.
  Every injected mailbox message is explicitly marked agent-origin, so a
  teammate can never mistake a relayed message for the human operator's own
  instruction or approval.
- Custom provider wrappers (`providers/README.md`) get one new file in the
  env contract for team turns: `PALLIUM_WORKFLOW_SESSION_FILE`. Read it at the
  start of your turn (empty means first turn ever); before you exit, overwrite
  it with whatever session/thread id your CLI needs to resume that exact
  conversation next time. Pallium never inspects the value, it only shuttles
  it between turns.

## Cost tracking caveat

Not every provider reports real spend for a team turn. claude teammates, and
wrapper teammates that report `cost_usd` via `PALLIUM_WORKFLOW_USAGE_FILE`,
are cost-tracked — their spend counts toward `--budget-usd`. **codex
teammates are never cost-tracked** (`codex exec` reports no usage/cost at
all, true of the regular non-team worker path too) — a codex-only team's
budget ceiling will not trigger on its own. `pallium team status` and `team
start --budget-usd` now say plainly which providers aren't tracked instead of
showing an indistinguishable $0.0000.
