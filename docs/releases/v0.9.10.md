# v0.9.10 — Agent Teams

A third way to orchestrate work: independent peer agents that coordinate with
each other directly.

## What's new

- **`pallium team`.** Spawn a lead plus independent named teammates that
  share a task board and a mailbox. Unlike a workflow's parallel fan-out,
  a team is built for work that benefits from peers reasoning independently
  and messaging each other — parallel review from different angles, a
  research-then-synthesize pattern, or adversarial debate.
- **Real persistent sessions.** Each teammate keeps its own ongoing
  conversation with its model across every turn — and that conversation
  survives even if the process steering the team is killed and reattached
  later.
- **No daemon required.** `pallium team run` advances a team until it
  converges, then exits; `pallium team attach` reconnects to a team in
  progress from anywhere, anytime.
- **Bring your own model.** Any provider can join a team through the same
  minimal wrapper contract Pallium already uses for workflow workers.

## Install

```bash
npm install -g pallium
pallium version
```
