# v0.9.11 — Loops

A fourth orchestration primitive: bounded cycles that keep working over time,
without a daemon.

## What's new

- **`pallium loop`.** A persistent, named cycle you (or a scheduler) advance
  one tick at a time. Each tick runs one bounded round and then stops — no
  background process, and no risk of runaway retries.
- **Knows when it's stuck.** Loops detect stagnation automatically and stop
  cleanly with a clear reason instead of spinning forever on a problem that
  isn't improving.
- **Budget-aware.** A loop can be given a spend ceiling per cycle and over
  its lifetime; going over either one stops it automatically.
- **A ready-made template.** `review-until-clean` drives review findings down
  to zero, one cycle at a time, out of the box.

## Install

```bash
npm install -g pallium
pallium version
```
