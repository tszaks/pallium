# v0.9.4

Pallium v0.9.4 makes dynamic workflows the primary orchestration surface for
large agentic coding tasks.

## Added

- Claude Ultracode-shaped workflow semantics documented in `PALLIUM_WORKFLOW.md`.
- Streaming `pipeline(items, stage1, stage2, ...)` stages, so fast items advance
  without waiting behind a stage-wide barrier.
- Workflow `log()` and `budget` script globals.
- Determinism guards for `Math.random()`, `Date.now()`, and argless `new Date()`
  to keep resume caching sound.
- Collection caps for `parallel()` and `pipeline()` to stop runaway scripts.
- Acceptance coverage that proves streaming pipeline behavior through the CLI.

## Changed

- Rewrote the README around Pallium as an agent control plane with repo memory,
  verification, sessions, dynamic workflows, API, MCP, and provider adapters.
- Tightened workflow gate approval so approving the wrong gate name fails instead
  of creating a phantom approval.
- Expanded workflow runtime tests for pipeline timing, stage arguments, item
  drops, deterministic guards, budget shape, and item limits.

## Verification

```bash
go test ./... -count=1
go vet ./...
go test -race ./...
bash scripts/workflow-verify.sh
scripts/workflow-acceptance.sh
pallium workflow audit --run-acceptance --json
```
