# v0.7.5 — Smarter test detection

This release teaches Pallium to recognize how a project actually runs its
tests, instead of guessing.

## What's new

- **Reads your repo's own test config.** Pallium now checks common project
  files (Makefile, pytest.ini, tox.ini, and similar) to figure out the right
  verification command instead of assuming one convention fits every repo.
- **Broader JavaScript and Python support.** Verification planning now covers
  more real-world project layouts in both ecosystems.

## Install

```bash
npm install -g pallium
pallium version
```
