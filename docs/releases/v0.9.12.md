# v0.9.12

Pallium v0.9.12 fixes four session-scoping bugs surfaced by real dogfooding: a shared-global-DB
query pattern that wasn't scoped to the calling session, branch, or repo. No new engine pillar in
this release — just the ground under the existing ones getting more solid.

## Fixed

- **Active task leak across branches**: `workflow preflight` could surface another session's
  in-flight task into an unrelated branch's result — `active_tasks` was keyed by `repo_id` alone,
  but one repo checkout is shared across branches and sessions. Now composite-keyed by
  `(repo_id, branch)`, scoped to the live git branch at read/write time, and ages out after 4
  hours so a stale row can never resurface.
- **Related-sessions bloat**: `workflow report <run-id>` always pulled in a `related_sessions`
  block from the *global* cross-repo session index — unrelated Codex/Claude transcripts from
  other repos bundled into a scoped run report. Now gated behind an opt-in `--related` flag.
- **Honest run status on partial failure**: a run whose top-level script rejected after some
  agents had already completed reported a flat `"failed"` status with a message that could
  render empty (a provider exiting nonzero with no captured stderr rendered as
  `workflow provider "x" failed: exit status 1: ` — nothing after the trailing colon). Two fixes:
  `SetRunStatus` now backstops so a failing run is never persisted with a blank error, and a run
  with at least one completed agent gets an honest `"completed_with_failures"` status instead of
  a flat `"failed"` that contradicts a still-usable `workflow report`. Every place that treats run
  status as an enum (GC eligibility, `workflow watch`'s terminal check, fleet/analytics status
  counts, loop tick error classification) now handles the new status consistently.
- **`workflow generate --cwd`**: `preflight` already had a `--cwd` flag; `generate` didn't, so
  `--save` and a relative `--output` silently resolved against Pallium's own process cwd instead
  of the target repo unless you `cd` first. Added `--cwd`, threaded through to the `--llm`
  provider dispatch too.

## Verification

```bash
node npm/scripts/test.js
go vet ./...
go test ./... -count=1 -race
bash scripts/workflow-verify.sh
pallium workflow audit --run-acceptance
```

Every fix above has a regression test verified by temporarily reverting the fix, confirming the
test fails, then restoring it.
