# v0.9.6

Pallium v0.9.6 hardens dynamic workflow semantics and prepares the repository
for wider open source use.

## Fixed

- Changed workflow agent caching to use deterministic call positions, so
  repeated prompts in loops spawn independently while resumes reuse the
  completed prefix.
- Preserved workflow agent and budget caps across resume.
- Made workflow decisions idempotent across resume.
- Resolved async `pipeline()` stages before continuing to the next stage.
- Changed parallel fan-out agent failures to return `null` for that item while
  preserving sibling results.
- Added local schema validation for cached and provider-returned structured
  workflow outputs.
- Removed the manual `workflow gate approve` command so workflow gates remain
  autonomous agent-verifier gates.
- Hardened resume and patch semantics from PR review: legacy agent cache
  migration, deterministic pipeline cache indexes, persisted resume caps,
  schema-failed patch suppression, whitespace-only API token rejection, and
  nullable schema validation.

## Hardened

- Added bearer-token authentication for `pallium workflow serve`, with token
  enforcement for non-local binds.
- Added release and npm publish checks that require the Git tag to match
  `npm/package.json`.
- Hardened the GitHub Packages publish workflow with the same release tag
  version check.
- Added CodeQL, OpenSSF Scorecard, Dependabot, issue templates, PR template,
  CODEOWNERS, contribution docs, support docs, code of conduct, and a security
  policy.
- Expanded workflow adversarial, resume, failure-injection, schema, API, and
  acceptance coverage.
