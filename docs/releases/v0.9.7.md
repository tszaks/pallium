# v0.9.7 — Workflows adopt your steering agent

Workflows now use whichever AI model is already driving your session, with
zero configuration.

## What's new

- **Automatic model adoption.** Run a workflow from inside Claude Code and
  its workers use Claude automatically — no configuration needed. Codex
  remains the fallback when no other agent is detected.
- **Built-in Claude support.** No wrapper script required to use Claude as a
  worker; Pallium talks to the `claude` CLI directly, with a locked-down
  sandbox by default.
- **Opt-in network access.** Workers get no network access unless a task and
  the run both explicitly allow it.

## Fixed

- **No more crashes on large prompts.** A large task description could
  previously overflow a system limit and crash the worker. Fixed.
- **A recurring test flake.** Closed a timing issue that occasionally caused
  a false failure in the console test suite.

## Install

```bash
npm install -g pallium
pallium version
```
