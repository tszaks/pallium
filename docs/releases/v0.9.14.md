# v0.9.14 — Dogfood fixes

Three real bugs found by using Pallium on real work, plus a fix that keeps
test runs from touching your real data.

## Fixed

- **A failing worker no longer takes down the whole batch.** One transient
  error inside a parallel batch of workers now only affects that one item,
  not the entire run.
- **Resuming a background run no longer crashes.** Resuming the same run in
  the background a second time used to fail outright. Fixed.
- **Test runs stay isolated.** A local database override meant for testing
  now consistently applies to every Pallium command, so test and dogfood
  runs can no longer accidentally write into your real session history.

## Install

```bash
npm install -g pallium
pallium version
```
