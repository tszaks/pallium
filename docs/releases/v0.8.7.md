# v0.8.7 — Handles multi-gigabyte transcripts

Closes out the large-transcript indexing work with a fix for the most
extreme case: multi-gigabyte Codex rollout files.

## Fixed

- **No more indexing failures on pathologically large files.** Oversized
  transcripts are now indexed as metadata-only records instead of being
  fully parsed into memory. Normal-sized transcripts are unaffected and are
  still fully indexed for messages, search, and semantic lookup.

## Install

```bash
npm install -g pallium
pallium version
```
