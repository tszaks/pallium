# v0.9.13 — See your own edits mid-run

Verification and review steps can now see a workflow's own in-progress edits,
and concurrent edit runs no longer collide.

## What's new

- **Mid-run visibility into your own edits.** A verification check, a
  quality gate, or a later editing step can now see earlier edits from the
  same run instead of the untouched original files.
- **Safe concurrent edits.** Running two edit workflows against the same repo
  at once no longer risks a race condition — a second run fails fast with a
  clear message instead of corrupting the first.

## Fixed

- **No more stuck runs after a rejected script.** A run that failed partway
  through could leave an orphaned worktree behind. Cleaned up automatically
  now.
- **Non-conflicting edits now compose correctly.** Two workers making
  separate changes to the same file used to fail outright instead of
  combining cleanly.

## Install

```bash
npm install -g pallium
pallium version
```
