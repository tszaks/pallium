# v0.9.5

Pallium v0.9.5 is a patch release for repo-memory accuracy and release safety.

## Fixed

- Fixed git history parsing so multiline commit bodies cannot be mistaken for
  changed file paths.
- Preserved multiline commit bodies while keeping changed-file lists separated
  by an explicit machine-readable delimiter.
- Added regression coverage for commit bodies that previously polluted
  `ChangedFiles`.
- Changed `gate()` from a human pause/resume checkpoint into an autonomous
  verifier-agent gate that approves or rejects workflow continuation.

## Added

- CI for pushes and pull requests, covering the npm wrapper test, `go vet ./...`,
  and `go test ./... -count=1`.
- Agent-gate acceptance coverage and tests for approved and rejected gate
  verdicts.

## Verification

```bash
node npm/scripts/test.js
go test ./internal/gitlog -count=1 -v
go test ./... -count=1
go vet ./...
```

## Install

```bash
npm install -g pallium
pallium version
```
