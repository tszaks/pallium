# Support

Use GitHub Issues for reproducible bugs and actionable feature requests.

Use GitHub Discussions, if enabled, for:

- Questions
- Ideas
- Workflow design discussion
- Provider integration discussion

Before opening a bug report, include:

- Pallium version from `pallium version --json`
- Install method
- Operating system
- Command run
- Expected result
- Actual result
- Relevant logs with secrets removed

Security reports should follow `SECURITY.md`, not public support threads.
